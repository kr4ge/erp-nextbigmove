# Impeccable Design Context — Next Big Move ERP

## Design Context

### Users
Operations teams, business owners, marketing leaders, and marketers managing e-commerce orders, integrations, analytics, and workflows across multiple POS stores (Pancake POS). They use this app daily for hours at a time on desktop (1920x1080 primary). Speed, data density, and clarity are critical, but KPI users also need motivation loops (streaks, badges, challenges, leaderboard) that reinforce consistent execution.

### Brand Personality
**Professional. Performance-Driven. Motivating.**

The app should feel like a trusted business operating system with healthy competitive energy. It should be structured and credible first, then motivational through KPI progress mechanics. Every visual decision should improve focus, accountability, and momentum.

Emotional goal: **Confident, Focused, Motivated** — users should trust the data and feel pushed to improve results.

### Aesthetic Direction

**Theme: White + Orange (Professional-Gamified Layer)**

Scope and rollout:
- **Phase 1 (now):** Apply gamified language to `dashboard.*` KPI surfaces only.
- **Phase 2 (forward):** Expand the same language to other modules in future updates.
- Keep current visual foundation; do not do a full visual reset.

- **Primary accent**: Orange (`#F97316` / Tailwind `orange-500`) — used sparingly for interactive elements only
- **Primary hover**: `#EA580C` (`orange-600`)
- **Primary subtle bg**: `#FFF7ED` (`orange-50`) — for active badges, selected states
- **Primary focus ring**: `orange-200` or `orange-100`
- **Backgrounds**: White (`#FFFFFF`) for cards, very light warm gray for page background
- **Text**: Slate-900 primary, slate-500 secondary, slate-400 tertiary
- **Borders**: Slate-200 (`#E2E8F0`)
- **Semantic colors**: Emerald for success, rose for error, amber for warning — never override with orange

**Gamified elements allowed (dashboard KPI views):**
- Streak indicators
- Achievement badges
- Challenge/target progress treatments
- Team/member leaderboard blocks

**Where orange appears:**
- Primary action buttons (Save, Sync, Submit)
- Active tab indicators and count badges
- Focus rings on inputs
- Selected/active navigation states
- Icon accents in section headers
- Link/action text
- Metric card icon backgrounds (default tone)
- Subtotal/price accents in order details

**Where orange does NOT appear:**
- Large background areas (keep white/slate)
- Semantic status indicators (success=green, error=red, warning=amber)
- Data table text (keep slate for readability)
- Neutral UI elements (borders, dividers, disabled states)
- Body text or paragraph content

**Theme mode**: Keep current mode behavior (light default with dark support). Gamification must work in the existing mode system, not replace it.

**Visual references** (take the *feel*, not the look):
- **Stripe Dashboard**: Data density, metric cards with context, single-accent-color discipline
- **Linear**: Ultra-clean spacing, information density without clutter, minimal color usage
- **Fitness/Productivity progress UIs**: clear streaks, level progress, and performance milestones without visual noise

**Anti-references**:
- NOT like Pancake POS (too colorful for our context)
- No childish game UI, mascot-heavy visuals, or cartoon effects
- No glassmorphism, gradient text, neon accents, or decorative illustrations
- No bouncy animations or excessive motion

### Design Principles

1. **Business-first gamification** — KPI gamification should improve behavior and accountability, not entertain. Reward mechanics must map directly to measurable business targets.

2. **Keep the current system, add motivation layers** — Preserve the existing white/orange/slate foundation and component language. Add gamified signals through streaks, badges, challenges, and leaderboard hierarchy.

3. **Progress must be instantly legible** — Team and member KPI states should be scannable in seconds: current value, target, status, and progress trend should always be visible.

4. **Hierarchy over novelty** — Always present KPI structure as: executive summary, then team performance, then member breakdown. Gamified treatments must reinforce this order.

5. **Compact but production-ready** — Maintain dense but breathable layouts across desktop breakpoints. Avoid oversized KPI widgets and heavy nested card stacks.

6. **Calm motion, strong feedback** — Use subtle transitions and micro-feedback; no flashy animation loops. Interactions should feel responsive and intentional.

### Technical Design System

**Font**: Inter (Google Fonts) — keep as-is, optimal for data-dense interfaces
**Base radius**: 0.5rem (8px), cards at rounded-2xl (16px), buttons at rounded-xl (12px)
**Shadows**: shadow-sm only, never shadow-md or larger for static elements
**Transitions**: 150-200ms ease, no bounce/spring/elastic
**Touch targets**: Minimum 32px height for desktop (not 44px — this is desktop-first)
**Number formatting**: `tabular-nums` for all financial data and quantities
**Icon sizes**: 3.5x3.5 (14px) in headers, 4x4 (16px) in buttons, 5x5 (20px) in navigation

**Section card pattern** (canonical):
```tsx
<section className="overflow-visible rounded-xl border border-slate-200 bg-white shadow-sm">
  <div className="flex items-center gap-2 border-b border-slate-100 bg-slate-50/80 px-3 py-2">
    <Icon className="h-3.5 w-3.5 text-orange-500" />
    <h4 className="text-xs font-semibold uppercase tracking-wide text-slate-700">TITLE</h4>
    <span className="ml-auto text-[10px] text-slate-500">metadata</span>
  </div>
  <div className="p-3">{content}</div>
</section>
```

**Button primary variant** (target):
```
bg-orange-500 hover:bg-orange-600 active:bg-orange-700 text-white
focus:ring-2 focus:ring-orange-200
```

**Active navigation** (target):
```
bg-orange-50 text-orange-700 border-l-2 border-orange-500
```

**Active tab** (target):
```
border-b-2 border-orange-500 text-orange-600
```

**MetricCard default tone** (target):
```
icon bg: bg-orange-50 text-orange-600
```
