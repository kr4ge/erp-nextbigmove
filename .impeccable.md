# Impeccable Design Context — Next Big Move ERP

## Design Context

### Users
Operations teams, business owners, and marketers managing e-commerce orders, integrations, analytics, and workflows across multiple POS stores (Pancake POS). They use this app daily for hours at a time on desktop (1920x1080 primary). Speed, data density, and clarity are critical — they process hundreds of orders and need to scan tables, update statuses, and monitor KPIs efficiently.

### Brand Personality
**Professional. Modern. Organized.**

The app should feel like a trusted business partner — confident, reliable, and smart. Not playful, not flashy. When users sit down to work, the interface should disappear and let the data speak. Every pixel should earn its place.

Emotional goal: **Confident & Reliable** — users should feel in control and trust the information on screen.

### Aesthetic Direction

**Theme: White + Orange (Subtle Accent)**

- **Primary accent**: Orange (`#F97316` / Tailwind `orange-500`) — used sparingly for interactive elements only
- **Primary hover**: `#EA580C` (`orange-600`)
- **Primary subtle bg**: `#FFF7ED` (`orange-50`) — for active badges, selected states
- **Primary focus ring**: `orange-200` or `orange-100`
- **Backgrounds**: White (`#FFFFFF`) for cards, very light warm gray for page background
- **Text**: Slate-900 primary, slate-500 secondary, slate-400 tertiary
- **Borders**: Slate-200 (`#E2E8F0`)
- **Semantic colors**: Emerald for success, rose for error, amber for warning — never override with orange

**Where orange appears:**
- Primary action buttons (Save, Sync, Submit)
- Active tab indicators and count badges
- Focus rings on inputs
- Selected/active navigation states
- Icon accents in section headers
- Link/action text
- Metric card icon backgrounds (default tone)
- Subtotal/price accents in order details

**Where orange does NOT appear:**
- Large background areas (keep white/slate)
- Semantic status indicators (success=green, error=red, warning=amber)
- Data table text (keep slate for readability)
- Neutral UI elements (borders, dividers, disabled states)
- Body text or paragraph content

**Theme mode**: Light mode default, dark mode as an option. Dark mode should invert intelligently — orange accent remains but adapts to darker contexts.

**Visual references** (take the *feel*, not the look):
- **Stripe Dashboard**: Data density, metric cards with context, single-accent-color discipline
- **Linear**: Ultra-clean spacing, information density without clutter, minimal color usage
- **Mercury**: Financial data presented with calm trustworthiness, restraint as a design feature

**Anti-references**:
- NOT like Pancake POS (too colorful, too playful for our professional context)
- No glassmorphism, gradient text, neon accents, or decorative illustrations
- No bouncy animations or excessive motion

### Design Principles

1. **Data density over decoration** — This is a business tool used for hours daily. Users need to see 12-15 table rows at 100% zoom. Every component should maximize information without sacrificing readability. No wasted space on decorative elements.

2. **One accent, infinite restraint** — Orange is the only accent color. It appears exclusively on interactive/actionable elements. If something has orange, it means "you can click this." Everything else is white, slate, or semantic (green/red/amber). Color is a signal, not decoration.

3. **Consistent section language** — Every content section uses the same card pattern: icon + uppercase title header bar (slate-50/80 bg), content below. This repetition builds spatial memory — users learn the layout once and navigate by instinct.

4. **Compact but breathable** — Use `text-xs` to `text-sm` for most content. `text-base` for section titles, never larger except page headers. Spacing is tight but consistent — `space-y-3`, `gap-2`, `p-3`. The goal is density, not cramming.

5. **Let the data be the hero** — No element should compete with the actual data. Borders are subtle (slate-200), shadows are minimal (shadow-sm), backgrounds are near-white. The numbers, names, and statuses are the visual interest.

### Technical Design System

**Font**: Inter (Google Fonts) — keep as-is, optimal for data-dense interfaces
**Base radius**: 0.5rem (8px), cards at rounded-2xl (16px), buttons at rounded-xl (12px)
**Shadows**: shadow-sm only, never shadow-md or larger for static elements
**Transitions**: 150-200ms ease, no bounce/spring/elastic
**Touch targets**: Minimum 32px height for desktop (not 44px — this is desktop-first)
**Number formatting**: `tabular-nums` for all financial data and quantities
**Icon sizes**: 3.5x3.5 (14px) in headers, 4x4 (16px) in buttons, 5x5 (20px) in navigation

**Section card pattern** (canonical):
```tsx
<section className="overflow-visible rounded-xl border border-slate-200 bg-white shadow-sm">
  <div className="flex items-center gap-2 border-b border-slate-100 bg-slate-50/80 px-3 py-2">
    <Icon className="h-3.5 w-3.5 text-orange-500" />
    <h4 className="text-xs font-semibold uppercase tracking-wide text-slate-700">TITLE</h4>
    <span className="ml-auto text-[10px] text-slate-500">metadata</span>
  </div>
  <div className="p-3">{content}</div>
</section>
```

**Button primary variant** (target):
```
bg-orange-500 hover:bg-orange-600 active:bg-orange-700 text-white
focus:ring-2 focus:ring-orange-200
```

**Active navigation** (target):
```
bg-orange-50 text-orange-700 border-l-2 border-orange-500
```

**Active tab** (target):
```
border-b-2 border-orange-500 text-orange-600
```

**MetricCard default tone** (target):
```
icon bg: bg-orange-50 text-orange-600
```
